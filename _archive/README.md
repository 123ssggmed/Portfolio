# Archive Policy

This repository previously collected many unrelated or draft artifacts in the root directory: old prompts, RTF plans, logs, pasted text, prospect CSVs, generated experiments, and multiple duplicate versions of portfolio documents.

The public portfolio now lives in `docs/` and intentionally links only to curated professional evidence. Material that is not linked from the site should be treated as archival or internal support material unless it is explicitly promoted into the public portfolio.

## Public-Facing Material

Use the website and the strongest PDF reports as the public layer:

- AI workflow and audit intake control
- Pricing and monetization architecture
- Embedded finance and fintech strategy
- AI-augmented diligence operating model
- Selected older PDF assets as secondary evidence

## Not Public-Facing By Default

Do not feature these categories in the public website without a separate review:

- Prospect CSVs and outreach lists
- Logs and debug files
- Pasted text dumps
- RTF plans and scratch documents
- Internal agent, persona, or workflow notes
- Prompt packs and execution instructions
- Duplicate DOCX/PDF/PPTX versions that dilute the main portfolio signal
- Sales-offer experiments unrelated to the portfolio narrative

## Cleanup Approach

The first cleanup pass is archive-first, not destructive. Existing files are not deleted. The immediate goal is to make the public-facing path professional while preserving source material for later manual review.

Future cleanup can move legacy files into structured archive folders or remove sensitive material from the public repository if needed.
